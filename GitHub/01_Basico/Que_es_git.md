# ¿Cuál es la diferencia entre Git y GitHub?

**Git** es un sistema de control de versiones distribuido. Se usa para llevar un historial de los cambios en archivos de un proyecto, permitiendo trabajar en equipo sin sobrescribir el trabajo de los demás.

**GitHub**, en cambio, es una plataforma en línea que permite almacenar repositorios Git en la nube. Ofrece herramientas colaborativas como issues, pull requests, y acciones automáticas (GitHub Actions).

| Característica | Git | GitHub |
|----------------|-----|--------|
| Tipo | Software de control de versiones | Plataforma en la nube |
| Uso | Local (en tu PC) | Remoto (en servidores) |
| Necesita internet | No | Sí |
| ¿Quién lo desarrolla? | Software libre, mantenido por la comunidad | Microsoft |

 *Git funciona sin GitHub, pero GitHub necesita Git para funcionar.*
